\doxysubsection{Detailed Description}
Container para armazenar e gerenciar reservas do sistema. 

Esta classe implementa um container complexo para armazenamento em memória de objetos do tipo \doxylink{class_reserva}{Reserva}. Utiliza uma estrutura hierárquica de mapas aninhados para permitir buscas eficientes por diferentes critérios.

Estrutura de dados\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{unordered\_map<int,\ unordered\_map<string,\ vector<Reserva*>>>}
\DoxyCodeLine{\ \ \ \ Chave\ externa:\ número\ \textcolor{keywordflow}{do}\ quarto\ (\textcolor{keywordtype}{int})}
\DoxyCodeLine{\ \ \ \ Chave\ interna:\ email\ \textcolor{keywordflow}{do}\ hóspede\ (\textcolor{keywordtype}{string})}
\DoxyCodeLine{\ \ \ \ Valor:\ vector\ de\ reservas\ (Reserva*)}

\end{DoxyCode}


Esta estrutura permite\+:
\begin{DoxyItemize}
\item Buscar todas as reservas de um quarto específico\+: O(1)
\item Buscar todas as reservas de um hóspede específico\+: O(n) onde n = número de quartos
\item Buscar reservas de um hóspede em um quarto\+: O(1)
\item Buscar reserva por código\+: O(n\texorpdfstring{$\ast$}{*}m) onde n = quartos, m = hóspedes
\end{DoxyItemize}

Características principais\+:
\begin{DoxyItemize}
\item Cada reserva possui código único (10 caracteres alfanuméricos)
\item Um quarto pode ter múltiplas reservas (em períodos diferentes)
\item Um hóspede pode ter múltiplas reservas (em quartos/períodos diferentes)
\item Validação de conflito de datas é feita externamente
\item Suporta operações de remoção em cascata (por quarto ou por hóspede)
\end{DoxyItemize}

Regras de negócio\+:
\begin{DoxyItemize}
\item Código\+: 10 caracteres alfanuméricos, único no sistema
\item Datas\+: chegada \texorpdfstring{$<$}{<} partida, formato DD-\/\+MMM-\/\+AAAA
\item Valor\+: calculado automaticamente baseado no período e diária
\item Não permite sobreposição de datas no mesmo quarto 
\end{DoxyItemize}
