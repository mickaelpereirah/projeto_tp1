\doxysubsection{Detailed Description}
Representa um valor monetário válido no sistema. 

A classe \doxylink{class_dinheiro}{Dinheiro} armazena valores monetários como inteiros para evitar erros de arredondamento típicos de ponto flutuante. O valor é armazenado em centavos (menor unidade monetária).

{\bfseries{FORMATO VÁ\+LIDO\+:}}
\begin{DoxyItemize}
\item Valor de 0,01 a 1.\+000.\+000,00
\item Armazenamento deve ser em inteiro para evitar erro de arredondamento
\end{DoxyItemize}

{\bfseries{REGRAS DE VALIDAÇÃO\+:}}
\begin{DoxyEnumerate}
\item Valor mínimo\+: 0,01 (1 centavo = 1 unidade inteira)
\item Valor máximo\+: 1.\+000.\+000,00 (100.\+000.\+000 centavos)
\item Armazenamento interno\+: valor em centavos como inteiro
\begin{DoxyItemize}
\item Exemplo\+: R\$ 10,50 → armazenado como 1050
\item Exemplo\+: R\$ 100,00 → armazenado como 10000
\end{DoxyItemize}
\end{DoxyEnumerate}

{\bfseries{EXEMPLOS\+:}}
\begin{DoxyItemize}
\item ✅ Válido\+: 0,01 (1 centavo)
\item ✅ Válido\+: 10,50 (R\$ 10,50)
\item ✅ Válido\+: 1000,00 (R\$ 1.\+000,00)
\item ✅ Válido\+: 1000000,00 (R\$ 1.\+000.\+000,00 -\/ valor máximo)
\item ❌ Inválido\+: 0,00 (abaixo do mínimo)
\item ❌ Inválido\+: -\/10,00 (valor negativo)
\item ❌ Inválido\+: 1000000,01 (acima do máximo)
\end{DoxyItemize}

{\bfseries{NOTA TÉ\+CNICA\+:}} O armazenamento em inteiro evita problemas como\+:
\begin{DoxyItemize}
\item 0.\+1 + 0.\+2 != 0.\+3 (erro de ponto flutuante)
\item Perda de precisão em operações monetárias
\item Arredondamentos incorretos
\end{DoxyItemize}

Para converter\+:
\begin{DoxyItemize}
\item double para centavos\+: valor\+\_\+double \texorpdfstring{$\ast$}{*} 100
\item centavos para double\+: valor\+\_\+centavos / 100.\+0 
\end{DoxyItemize}
