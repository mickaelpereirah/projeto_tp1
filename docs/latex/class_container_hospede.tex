\doxysubsection{Detailed Description}
Container para armazenar e gerenciar hóspedes do sistema. 

Esta classe implementa um container para armazenamento em memória de objetos do tipo \doxylink{class_hospede}{Hospede}. Utiliza um unordered\+\_\+map com o email do hóspede como chave, garantindo acesso rápido O(1) e unicidade de emails no sistema.

O container é responsável por\+:
\begin{DoxyItemize}
\item Armazenar hóspedes de forma única (um email = um hóspede)
\item Permitir busca eficiente por email
\item Gerenciar inclusão, remoção e atualização de hóspedes
\item Manter a integridade dos dados (não permite emails duplicados)
\item Listar todos os hóspedes cadastrados no sistema
\end{DoxyItemize}

Diferente dos containers de \doxylink{class_hotel}{Hotel} e \doxylink{class_quarto}{Quarto}, os hóspedes não são vinculados diretamente a um hotel específico no container. A associação com hotéis é feita através das reservas.

Regras de negócio\+:
\begin{DoxyItemize}
\item \doxylink{class_email}{Email}\+: formato válido, único no sistema (chave primária)
\item \doxylink{class_nome}{Nome}\+: 5-\/20 caracteres, primeira letra maiúscula
\item Endereço\+: 5-\/30 caracteres
\item Cartão\+: 16 dígitos com validação de Luhn
\end{DoxyItemize}

IMPORTANTE\+: O container NÃO é responsável por liberar a memória dos objetos. A responsabilidade de delete dos ponteiros é de quem criou os objetos.
